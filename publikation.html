Testen der Seite

Bevor die Seite auf den Webserver geladen wird, sollte sie ausgiebig in verschiedenen Browsern auf unterschiedlichen Endgeräten getestet werden
Die höchste Wahrscheinlichkeit für eine korrekte Darstellung in allen Browsern hat fehlerfreier Code. Ob eine Webseite syntaktisch korrekt ist, kann mit einem HTML-Validator validator.w3.org bzw. mit einem CSS-Validator jigsaw.w3.org/css-validator getestet werden.

Webspace

Die fertige Website muss vom lokalen Rechner auf einen Webserver geladen werden, um im Internet verfügbar zu sein.
Die ZEDAT bietet Universitätsangehörigen die Möglichkeit eine eigene Homepage auf den ZEDAT-Webserver Komma zu laden www.zedat.fu-berlin.de/Userpage.
Aber auch private Anbieter wie Hosteurope, Domain Factory, Strato und 1und1 stellen Webspace zur Verfügung, den Nutzer für ihre Webseiten mieten können. Die großen Anbieter sind preiswert, bieten jedoch meist wenig Kundenservice.
Es gibt auch kostenlose Anbieter, wie Hostinger, Bplaced oder das Baukastensystem Jimdo. Kostenlose Angebote finanzieren sich über Werbung und dienen oft der Neukundengewinnung. Für professionelle Projekte sind sie aufgrund ihrer schlechteren Performance nicht ratsam.
Bei kommerziellen Anbietern wird der Webspace mit einem gewählten Domainnamen verbunden. Ob die Domain noch verfügbar ist, kann beispielsweise bei Check Domain geprüft werden.

Upload

Der Upload der fertigen Site auf den Remote-Webserver erfolgt über SFTP. SFTP steht für Secure File Transfer Protocol und bezeichnet ein Übertragungsprotokoll, mit dem Rechner Dateien austauschen können. Zum Übertragen der Dateien kann beispielsweise der FTP-Client FileZilla verwendet werden. FileZilla ist ein Open Source FTP Programm, das man unter folgender Adresse herunterladen kann: http://www.filezilla.de.

FileZilla Konfiguration
Serverlogin.zedat.fu-berlin.de
Benutzernamezedatbenutzername (klein geschrieben !)
Passwordzedatpasswort
Port22 (SFTP Port)

Verzeichnisse werden auf dem Webserver übertragen, indem sie doppelt angeklickt oder mit gedrückter Maustaste von der linken in die rechte Fensterhälfte gezogen werden.

Testen der Site im Internet

Um die übertragene Website online zu testen, muss die folgende Adresse in den Browser eingegeben werden:

Syntax
http://userpage.fu-berlin.de/zedatbenutzername/unterverzeichnis_von_public_html/dateiname_mit_endung

Beispiel
http://userpage.fu-berlin.de/max/web1/publikation.html

Hinweis: Dateien, die über das Internet zugänglich sein sollen, müssen auf dem Webserver im Verzeichnis public_html liegen. Wird kein Dateiname angegeben sucht der Webserver automatisch nach einer Datei namens index.html. Die Startseite sollte daher index.html heißen und im oder in einem Unterverzeichnis von public_html liegen.

Spätere Änderungen an der Website

Spätere Änderungen werden zunächst an der lokalen Kopie der Site vorgenommen. Anschließend müssen die geänderten Dateien mithilfe von FileZilla auf dem Server aktualisiert werden.
